This guide walks you from a fresh clone to a running FreeRTOS application. The fastest way to evaluate FreeRTOS is using the Windows or POSIX simulator — no hardware required.

## Prerequisites

- A C99-compatible compiler (GCC, IAR, Keil/ARMCC, or MSVC)
- Git with submodule support
- For Windows simulator: Visual Studio 2019+ or MinGW
- For POSIX simulator: GCC on Linux or macOS

<Steps>
  <Step title="Clone the repository">
    Always clone with `--recurse-submodules` — the kernel and all libraries are Git submodules:

    <CodeGroup>
    ```bash HTTPS
    git clone https://github.com/FreeRTOS/FreeRTOS.git --recurse-submodules
    ```
    ```bash SSH
    git clone git@github.com:FreeRTOS/FreeRTOS.git --recurse-submodules
    ```
    </CodeGroup>

    <Warning>
      Downloading the ZIP from GitHub does **not** include submodule contents. You must use `git clone`.
    </Warning>
  </Step>

  <Step title="Open a demo project">
    Navigate to a demo matching your target environment:

    | Environment | Path |
    |-------------|------|
    | Windows (MSVC) | `FreeRTOS/Demo/WIN32-MSVC/` |
    | Windows (MinGW) | `FreeRTOS/Demo/WIN32-MingW/` |
    | Linux/macOS (POSIX) | `FreeRTOS/Demo/Posix_GCC/` |
    | QEMU ARM Cortex-M3 | `FreeRTOS/Demo/CORTEX_LM3S6965_GCC_QEMU/` |
    | QEMU RISC-V | `FreeRTOS/Demo/RISC-V_RV32_QEMU_VIRT_GCC/` |
    | STM32F407 (Keil/GCC) | `FreeRTOS/Demo/CORTEX_M4F_STM32F407ZG-SK/` |

    Each demo is pre-configured and ready to build. Open the project file in your IDE or run `make`/`cmake` from the demo directory.
  </Step>

  <Step title="Review FreeRTOSConfig.h">
    Every FreeRTOS project needs a `FreeRTOSConfig.h`. The demos include a pre-configured one. Key settings to know:

    ```c FreeRTOSConfig.h
    /* Scheduler behavior */
    #define configUSE_PREEMPTION              1
    #define configUSE_TIME_SLICING            1

    /* Tick rate — 1000 Hz = 1ms tick */
    #define configTICK_RATE_HZ               ( ( TickType_t ) 1000 )

    /* CPU clock in Hz (set for your hardware) */
    #define configCPU_CLOCK_HZ               ( ( unsigned long ) 25000000 )

    /* Task priorities */
    #define configMAX_PRIORITIES             ( 5 )

    /* Memory */
    #define configTOTAL_HEAP_SIZE            ( ( size_t ) ( 75 * 1024 ) )
    #define configMINIMAL_STACK_SIZE         ( ( unsigned short ) 128 )

    /* API inclusions */
    #define INCLUDE_vTaskDelay               1
    #define INCLUDE_vTaskDelayUntil          1
    #define INCLUDE_vTaskDelete              1
    #define INCLUDE_vTaskSuspend             1
    ```
  </Step>

  <Step title="Write your first tasks">
    Replace the demo's main.c with a minimal two-task application:

    ```c main.c
    #include "FreeRTOS.h"
    #include "task.h"
    #include "queue.h"

    #define QUEUE_LENGTH    5
    #define ITEM_SIZE       sizeof( uint32_t )

    static QueueHandle_t xQueue;

    /* Producer task: sends incrementing values to the queue */
    static void vProducerTask( void *pvParameters )
    {
        uint32_t ulValueToSend = 0;

        for( ;; )
        {
            xQueueSend( xQueue, &ulValueToSend, portMAX_DELAY );
            ulValueToSend++;
            vTaskDelay( pdMS_TO_TICKS( 200 ) );
        }
    }

    /* Consumer task: receives values from the queue */
    static void vConsumerTask( void *pvParameters )
    {
        uint32_t ulReceivedValue;

        for( ;; )
        {
            if( xQueueReceive( xQueue, &ulReceivedValue, portMAX_DELAY ) == pdPASS )
            {
                /* Process the received value */
            }
        }
    }

    int main( void )
    {
        /* Create a queue to hold 5 uint32_t values */
        xQueue = xQueueCreate( QUEUE_LENGTH, ITEM_SIZE );

        configASSERT( xQueue != NULL );

        /* Create tasks */
        xTaskCreate( vProducerTask, "Producer", configMINIMAL_STACK_SIZE, NULL, 2, NULL );
        xTaskCreate( vConsumerTask, "Consumer", configMINIMAL_STACK_SIZE, NULL, 1, NULL );

        /* Start the scheduler — this never returns */
        vTaskStartScheduler();

        /* Should never reach here */
        for( ;; );
    }
    ```
  </Step>

  <Step title="Build and run">
    Build the project using your toolchain. For the POSIX demo on Linux:

    ```bash
    cd FreeRTOS/Demo/Posix_GCC
    make
    ./build/posix_demo
    ```

    For the Windows MSVC demo, open `WIN32.sln` in Visual Studio and press **F5**.
  </Step>
</Steps>
